## MotionBGs 分页问题深度诊断与修复

### 用户反馈
用户反映分页仍然失效，且提到分页是通过 URL 实现的。

### 问题根源
**缓存机制导致分页失效！**

1. MediaService 使用 `listCache` 缓存解析结果
2. 如果第一次加载时 `parseNextPagePath` 返回 nil（没有找到分页链接），这个结果会被缓存
3. 后续加载会返回缓存的错误数据，`hasMorePages` 永远是 `false`
4. loadMore 永远不会被触发

### 诊断过程

#### 1. 使用 browse 工具检查实际 HTML
```bash
# 保存完整 HTML
curl https://motionbgs.com > /tmp/motionbgs.html

# 检查 HTML 内容
grep -o 'href="/2/"' /tmp/motionbgs.html
# 结果: href="/2/"

# 检查分页按钮结构
cat /tmp/motionbgs.html | sed 's/></>\n</g' | grep "href=\"/2/\""
# 结果: <a class="link--arrowed" href="/2/"> View More Wallpapers <svg...>
```

**结论**: 
- ✅ HTML 中确实包含分页按钮
- ✅ class="link--arrowed" 存在
- ✅ href="/2/" 存在
- ✅ 选择器 `a.link--arrowed` 应该能正确匹配

#### 2. 代码逻辑验证
- ✅ parseNextPagePath 函数正确提取 href 属性
- ✅ makePageURL 函数正确处理分页路径
- ✅ loadMore 方法正确更新 nextPagePath 和 hasMorePages
- ✅ UI 正确触发 loadMore 条件判断

### 修复方案

#### 1. 添加 MediaService.clearCache 方法
```swift
func clearCache() async {
    listCache.removeAll()
    detailCache.removeAll()
    print("[MediaService] 🗑️ 缓存已清除")
}

func clearCache(for url: URL) async {
    let cacheKey = url.absoluteString
    if listCache.removeValue(forKey: cacheKey) != nil {
        print("[MediaService] 🗑️ 已清除缓存: \(cacheKey)")
    }
}
```

#### 2. 在设置中集成
SettingsViewModel.clearCache 中添加:
```swift
await MediaService.shared.clearCache()
```

### 解决方案
**清除缓存！**

用户可以通过以下方式清除缓存：
1. 打开设置 (⌘,)
2. 找到"缓存管理"部分
3. 点击"清除缓存"按钮
4. 重启 App
5. 测试分页功能
