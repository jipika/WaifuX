## 编译错误修复 (2026-03-30 深夜)

### 发现的问题
1. **字符串字面量错误**: AnimeParser.swift 第37行 `"========"=` 应为 `"=========="`
2. **重复方法声明**: HTMLParser.swift 中同时存在:
   - `nonisolated func extractText/extractAttr` (简化版)
   - `private func extractText/extractAttr` (完整版)
   - Swift 不允许同名方法的访问级别不同导致的重复声明

### 解决方案
1. 修正字符串字面量
2. 重命名 HTMLParser 的简化方法:
   - `extractText` → `simpleExtractText`
   - `extractAttr` → `simpleExtractAttr`
3. 更新 AnimeParser.swift 中的所有调用

- **提交哈希**: `1b15e2f`
- **已推送到远程**: `https://github.com/jipika/WallHaven`
- **构建状态**: ✅ BUILD SUCCEEDED

### 关键代码修改

**HTMLParser.swift**:
```swift
// 重命名简化方法,避免与 private 方法冲突
nonisolated func simpleExtractText(element: Element, selector: String) throws -> String?
```

## 集成 Kazumi 规则仓库 (2026-03-30 深夜)

### 用户需求
用户发现 Kazumi 有实时更新的规则仓库,要求直接使用 Kazumi 的规则源。

### Kazumi 规则仓库分析
- **仓库地址**: `https://raw.githubusercontent.com/Predidit/KazumiRules/main/`
- **索引文件**: `index.json`
- **规则数量**: 16+ 个动漫源
- **更新频率**: 2026 年最新更新
- **格式**: XPath 格式 (api: 1 或 5)

### Kazumi 规则格式特点
```json
{
  "name": "AGE",
  "version": 1.5,
  "api": 1,
  "searchURL": "https://www.agedm.io/search?query=@keyword",
  "searchList": "//div[2]/div/section/div/div/div/div",
  "searchName": "//div/div[2]/h5/a"
}
```

**关键差异**:
1. **API 版本**: 整数 `1` 或 `5` (不是字符串 `"1"`)
2. **占位符**: `@keyword` (不是 `{keyword}`)
3. **选择器**: **XPath** (不是 CSS Selector)

### 实现方案
创建了 `KazumiRuleLoader.swift`:
- 从 Kazumi 规则仓库加载规则
- 自动转换 Kazumi 格式到本地 AnimeRule 格式
- 缓存已加载的规则
- 支持规则索引和单个规则加载

### AnimeRuleStore 更新
添加了双源支持:
- **用户规则仓库**: `jipika/WallHaven-Profiles`
- **Kazumi 规则仓库**: `Predidit/KazumiRules`
- 可通过 `switchRuleSource()` 切换
- 默认使用 Kazumi 规则

### 新增文件
- `/Services/KazumiRuleLoader.swift` - Kazumi 规则加载器

### 规则源对比

| 规则源 | 数量 | 格式 | 更新频率 |
|--------|------|------|----------|
| 用户仓库 | 少量 | CSS Selector | 不定期 |
| Kazumi | 16+ | XPath | 活跃更新 |
