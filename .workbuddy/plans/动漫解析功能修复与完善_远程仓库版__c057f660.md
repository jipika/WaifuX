## 用户需求分析

**核心问题**: 用户的动漫规则仓库使用 `api=1` 的简化 CSS Selector 格式,但项目解析逻辑存在问题,导致动漫列表和详情页无法加载。

**用户需求**:

1. 修复动漫列表加载功能
2. 修复动漫详情页解析功能  
3. 确保完全兼容远程规则仓库格式
4. 完成后将修改推送到远程仓库

**关键约束**:

- 只使用远程规则仓库
- 不需要本地配置文件支持
- 必须推送到远程仓库: `https://github.com/jipika/WallHaven` (私有)

**远程仓库规则格式** (api=1):

```
{
  "id": "age",
  "api": "1",
  "searchURL": "https://www.agedm.io/search?query={keyword}",
  "searchList": "a[href*='/detail/']",
  "searchName": "a[href*='/detail/']",
  "searchCover": "img",
  "searchDetail": "a[href*='/detail/']"
}
```

## 产品概述

修复 WallHaven 应用的动漫解析功能,确保动漫列表和详情页能够正确加载,并推送到远程仓库。

## 核心功能

1. CSS Selector 解析: 支持 `api=1` 格式的规则解析
2. 占位符处理: 正确处理 `{keyword}` 占位符
3. 相对路径转换: 将相对 URL 转为绝对路径
4. 错误处理: 完善的解析错误处理和日志
5. Git 推送: 完成修改后推送到远程仓库

## 技术栈

- **语言**: Swift
- **解析库**: SwiftSoup (HTML 解析)
- **架构**: MVVM + Actor 并发模型
- **版本控制**: Git

### 数据流

```
用户搜索请求
    ↓
AnimeViewModel.search(query: "进击的巨人")
    ↓
AnimeParser.searchWithRule()
    ↓
URL 构建: searchURL.replacingOccurrences(of: "{keyword}", with: encodedQuery)
    ↓
fetchHTML() → 下载网页 HTML
    ↓
parseSearchResultsV1() → CSS Selector 解析
    ├─ document.select("a[href*='/detail/']") → 列表元素
    ├─ element.select("img").attr("src") → 封面
    ├─ element.select("a[href*='/detail/']").attr("href") → 详情链接
    └─ makeAbsoluteURL() → 相对路径转绝对路径
    ↓
返回 AnimeSearchResult[]
    ↓
显示动漫列表
```
